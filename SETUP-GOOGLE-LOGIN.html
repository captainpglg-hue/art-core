Activer « Continuer avec Google »

Le code est déjà en place dans art-core et pass-core. État au 4 mai 2026 : Supabase reactivé, Site URL et 4 Redirect URLs configurées. Il reste 2 étapes bloquées sur des actions humaines (2FA Google + création Client ID).

! Avertissement sécurité

Dans Supabase → Auth → Providers → Google, j'ai vu que les champs Client IDs et Client Secret ont déjà été remplis avec des valeurs invalides :

  • Client IDs = nom du projet Supabase (pas un Client ID OAuth)
  • Client Secret = chaîne qui ressemble à un mot de passe personnel (pas un secret OAuth Google)

Le toggle « Enable Sign in with Google » est resté sur OFF, donc rien ne fonctionne — c'est juste à nettoyer. Si la chaîne dans Client Secret est un mot de passe que tu utilises ailleurs (Gmail, autres comptes), change-le. Un secret stocké dans la config Supabase est lisible par toute personne qui a accès au dashboard.

À l'étape 2 ci-dessous, tu remplaceras ces deux champs par les vraies valeurs venant de Google Cloud Console.

1 Google Cloud Console action user obligatoire

Bloqueur identifié : Google Cloud demande maintenant la 2FA sur ton compte Google avant tout accès à la console. C'est une action de sécurité personnelle — je ne peux pas la faire à ta place. Une fois la 2FA activée, tu pourras créer le Client ID OAuth.

Créer un Client ID OAuth 2.0 que Supabase utilisera pour parler à Google.

  1. Ouvrir Google Cloud Console (bouton ci-dessous).
  2. Crée un projet (ou réutilise Art-Core s'il existe).
  3. Va dans APIs & Services → OAuth consent screen. Type External. Renseigne nom de l'app (« Art-Core »), email support, domaines autorisés (art-core.app, pass-core.app, supabase.co).
  4. Puis APIs & Services → Credentials → + CREATE CREDENTIALS → OAuth client ID. Type Web application.
  5. Dans Authorized redirect URIs, colle l'URL ci-dessous :
https://umgskpgkcluberbvoamm.supabase.co/auth/v1/callback

Optionnel — pour tester aussi en local, ajoute aussi ces URIs :

http://localhost:3000/auth/callback
http://localhost:3001/auth/callback
  1. Clique Create. Une popup s'ouvre avec Client ID et Client Secret. Garde cette fenêtre ouverte — tu vas coller les deux à l'étape 2.
Ouvrir Google Cloud Console → Écran de consentement

2 Supabase Dashboard URL Config déjà fait ✓

Déjà fait pour toi : Site URL = https://art-core.app + 4 Redirect URLs (art-core.app/**, pass-core.app/**, localhost:3000/**, localhost:3001/**).

Quand tu auras le Client ID/Secret de l'étape 1 :

  1. Ouvre Supabase → Auth → Providers (bouton ci-dessous).
  2. Clique sur Google dans la liste (actuellement Disabled).
  3. Vide les champs Client IDs et Client Secret pré-remplis (valeurs invalides).
  4. Colle ton vrai Client ID dans Client IDs.
  5. Colle ton vrai Client Secret dans Client Secret (for OAuth).
  6. Active le toggle Enable Sign in with Google.
  7. Clique Save.
Ouvrir Supabase Auth Providers →

3 Déployer le code en prod 5 min

Le bouton existe déjà dans les fichiers locaux mais n'est pas encore en prod. Push sur main pour que Vercel auto-deploy.

Dans un terminal, depuis ~/Desktop/Projets/art-core :

git status

Vérifie que les fichiers app/auth/callback/route.ts et components/auth/GoogleSignInButton.tsx apparaissent.

git add -A && git commit -m "feat(auth): add Google OAuth login on art-core and pass-core"
git push origin main

Vercel va builder et déployer en ~2 minutes. Tu peux suivre sur le dashboard :

Vercel Dashboard →
